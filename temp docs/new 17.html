Media Content

1. When media type=stream, the input fields will be: Protocol (srt, udp, hls), srt mode, interface, ip address, port
2. When media type=local generated content, the file type dd will have values: video/audio test patterns, null PID transport stream files and digital signage.
if video, audio test patterns get selected, other fields will be disabled; new field for "duration" will show up for hours and minutes and seconds; these will be generated by gstreamer ; If null PID transport stream files is selected, then tbd. If digital signage is selected, then tbd
3. When any input file is saved, ts will be generated and these ts become the input to tsswitch whose output will be stream or recordings.
4. During the import media files need to be re-encoded to specified by the user resolution and framerate and re-muxed as Transport stream (.ts) files using gstreamer. When imported the media shall be converted to either h.264 video and aac audio with available resolution options for h.264 are 480p, 720p and 1080p, audio will always be stereo, 192k bitrate, 48kHz. Video bit-rate and GOP shall be specified by the user. GOP shall give the options to specify I-frame interval and B-frame count.
5. After the file is converted to ts we shall run it through TSDuck in order to stuff it to constant bitrate, fix continuity errors, resync, restamp the pcr and make unified PSI/SI tables.
